Ping-ris Wheel

An Interactive LED Ferris Wheel Gaming Platform Combining Pong & Tetris

üéØ Project Overview

The Ping-ris Wheel is a unique interactive art installation that transforms a 1.2-meter diameter Ferris wheel into a dynamic gaming platform. By mounting a 32√ó32 RGB LED matrix display on the rotating structure and implementing real-time game physics that account for rotation and gravity, this project creates an engaging multiplayer gaming experience unlike traditional flat-screen displays.

The name "Ping-ris" combines Pong (the classic arcade game) and Tetris, as the platform supports both games with physics adjusted for the circular, rotating display. Players control their games using wireless controllers while the wheel rotates, creating a challenging and visually spectacular experience. The project showcases integration of mechanical design, LED matrix control, real-time embedded programming, and game physics adaptation.

Ping-ris Wheel in Operation

1.2m diameter Ferris wheel with 32√ó32 RGB LED matrix display.

The completed Ping-ris Wheel features a 1.2-meter diameter structure with six evenly-spaced gondolas and a central 32√ó32 RGB LED matrix (1024 individually addressable LEDs). The wheel rotates continuously at variable speeds from 0-12 RPM via a DC gear motor with encoder feedback for precise position tracking. The LED matrix displays classic arcade games (Pong and Tetris) with game physics dynamically adjusted to account for the rotating reference frame and effective gravity vector that changes with wheel position. Structural components are powder-coated steel for weather resistance, enabling both indoor and outdoor installations. The entire system draws 120W at peak brightness and supports wireless controller connectivity for up to four simultaneous players.

Mechanical Structure & Fabrication

Steel frame construction with precision hub and spoke design.

The Ferris wheel structure was fabricated from 1-inch square steel tubing for the rim and spokes, with a custom-machined aluminum hub providing the central mounting point and motor attachment. The six-spoke radial design distributes loads evenly and provides mounting points for gondolas at 60¬∞ intervals. Critical design considerations included: balancing the structure by placing the heavy LED matrix opposite the motor and electronics gondola, ensuring sufficient rigidity to prevent wobble at maximum RPM (12), and maintaining clearances for 360¬∞ rotation without cable interference. The rim was constructed in six curved segments TIG-welded together, with precision jigs ensuring true circular geometry. All structural components were powder-coated matte black for a professional appearance and corrosion resistance. Total structural weight: 18kg, carefully balanced to minimize vibration.

üí° LED System

LED Matrix Display System

32√ó32 RGB addressable LED matrix with real-time game rendering.

The display uses four 16√ó16 RGB LED matrix panels (WS2812B-based) arranged in a 32√ó32 configuration, totaling 1024 individually addressable LEDs. Each LED contains an integrated driver chip enabling independent color control (16.7 million colors) via a single data line. The panels are mounted in a custom aluminum frame with a frosted diffuser to soften individual pixel visibility and improve viewing angles. Power distribution required careful design‚Äîat maximum brightness (white), the entire display draws 60A at 5V. This is managed through: a high-capacity 5V 24A power supply, multiple parallel power injection points across the matrix to prevent voltage drop, and programmatic brightness limiting to keep average current draw below 20A during gameplay. Data is driven by a Teensy 4.1 microcontroller (600 MHz ARM Cortex-M7) running the FastLED library, capable of achieving 120 FPS refresh rates for smooth animation even during wheel rotation.

üéÆ Software

Rotation-Aware Game Physics

Custom game engines accounting for rotating reference frame.

The most technically challenging aspect was adapting classic game physics to work correctly on a rotating display. For Pong: the ball's trajectory must account for Coriolis effects in the rotating frame‚Äîwhen the wheel rotates, the ball's apparent path curves. This required implementing physics that calculates ball position in both the rotating frame (for display) and inertial frame (for true motion), then applies appropriate coordinate transformations. For Tetris: gravity direction changes continuously as the wheel rotates‚Äîwhen the LED matrix is at the top of the rotation, "down" points toward the hub; at the side, "down" points radially. The game logic uses encoder feedback to determine current wheel orientation and applies gravity vector accordingly. Additional features include: difficulty scaling (faster rotation = harder gameplay), score tracking across rotations, and game state persistence through full wheel revolutions. All game logic runs at 60 FPS independent of display refresh rate to ensure consistent physics behavior.

Wireless Controller System

Custom wireless controllers supporting 4-player simultaneous gameplay.

Player control is handled through four custom-built wireless controllers communicating via 2.4 GHz RF (using nRF24L01+ modules). Each controller features: a joystick for directional input, action buttons for game-specific controls (ball serve in Pong, rotate/drop in Tetris), RGB LED status indicators showing controller pairing and battery level, and a rechargeable LiPo battery providing 8+ hours of continuous use. The wireless communication protocol implements: low-latency polling (1000 Hz) to ensure responsive controls, automatic controller pairing via unique IDs, and redundant packet transmission to prevent input drops. The base station (mounted on the wheel structure) receives all controller inputs and forwards them to the Teensy microcontroller. Controller enclosures were 3D printed in ergonomic designs with rubberized grips. The system supports hot-swapping controllers during gameplay, allowing players to join or leave without interrupting others.

Public Exhibition & User Experience

Successfully demonstrated at campus events with hundreds of players.

The Ping-ris Wheel has been showcased at multiple university events and public demonstrations, engaging over 500 players across various installations. User testing revealed fascinating insights: players initially struggled with the rotating reference frame but quickly adapted after 30-60 seconds of play. The constantly changing gravity direction in Tetris proved more challenging than anticipated, with even experienced Tetris players needing practice to master the rotation-aware gameplay. Most popular mode: competitive 2-player Pong where rotation speed increases with each rally, creating increasingly chaotic physics. The installation proved highly photogenic, with the illuminated spinning wheel creating striking visual effects in evening demonstrations. Lessons learned include: the importance of clear on-screen instructions (added an attract mode with gameplay tutorials), need for variable difficulty settings to accommodate skill ranges, and value of real-time score displays visible to spectators. Future enhancements planned include additional game modes, online leaderboards, and smartphone controller support via Bluetooth.

Concept Development & Physics Analysis

Concept Design Physics Modeling Rotating Frames Feasibility Study

üí° Key Insight: Initial concept simply displayed static images on a rotating LED matrix. However, analyzing the physics revealed that with position tracking, we could create games where the rotation itself becomes part of the gameplay mechanics. Realizing that gravity direction (from the player's perspective) changes continuously as the wheel rotates led to the breakthrough idea of implementing rotation-aware game physics where the challenge scales with rotation speed. This transformed the concept from passive art installation into interactive gaming platform.

Structural Design & Fabrication

Steel Fabrication Welding Load Analysis Balancing

üí° Key Insight: First structural prototype used a four-spoke design which proved insufficient‚Äîthe structure flexed noticeably during rotation causing LED matrix misalignment. Switching to six spokes dramatically improved rigidity with minimal weight penalty. More critically, initial assembly had severe vibration at 8 RPM‚Äîdiscovered this was due to LED matrix weight creating imbalance. Solving this required strategic placement of battery packs and electronics as counterweights, followed by dynamic balancing with the wheel spinning. The final balanced structure runs smoothly across entire 0-12 RPM range.

LED Matrix Integration & Power Management

WS2812B LEDs Power Distribution Current Management FastLED Library

üí° Key Insight: Initial LED wiring used single power injection point at corner‚Äîthis caused severe brightness fall-off across the matrix and warm LEDs at the far end. Measuring voltage revealed 0.8V drop across 1 meter of power bus at 15A. Solution required multiple power injection points (one per 16√ó16 panel) with heavy gauge wire (12 AWG) forming a power distribution grid. Additionally, discovered that enabling all LEDs at full white would draw 60A‚Äîway beyond power supply capacity. Implemented intelligent brightness limiting that monitors total current draw and dynamically adjusts per-pixel brightness to stay within 24A limit while maintaining good visual appearance.

Game Physics & Rotation-Aware Algorithms

C++ Programming Coordinate Transforms Real-time Systems Game Logic

üí° Key Insight: First Pong implementation simply drew the game on rotating display‚Äîbut ball trajectory looked wrong, it curved unnaturally. The issue: wasn't accounting for the rotating reference frame. Ball moves in straight lines in inertial frame, but appears to curve when viewed from rotating frame (Coriolis effect). Solution required implementing dual coordinate systems: calculating physics in inertial frame, then transforming to rotating frame for display. For Tetris, the big challenge was dynamically rotating gravity‚Äîtried applying gravity relative to wheel position but pieces moved erratically during fast rotation. Final solution: smooth gravity vector interpolation that gradually rotates as wheel turns, giving players predictable piece motion even at 12 RPM.

Wireless Controllers & Multiplayer System

RF Communication nRF24L01+ Protocol Design 3D Printing

üí° Key Insight: Initial controller design used Bluetooth for simplicity‚Äîbut Bluetooth latency (50-100ms) made gameplay feel sluggish, especially during fast wheel rotation where input timing is critical. Switching to 2.4 GHz RF with nRF24L01+ modules reduced latency to <5ms, dramatically improving responsiveness. However, discovered packet loss issues when multiple controllers transmitted simultaneously. Solution: implemented time-division multiplexing where each controller transmits in designated time slots, coordinated by base station. This eliminated collisions while maintaining low latency. Also learned that ergonomic controller design matters‚Äîearly 3D printed enclosures were uncomfortable for extended play; final iteration includes rounded edges, rubberized grips, and thumb-rest contouring based on user feedback.

User Testing & Public Demonstrations

User Experience Public Installation Feedback Analysis Feature Iteration

üí° Key Insight: Public demonstrations revealed surprising user behavior: first-time players struggled significantly more than expected with the rotating gameplay, often taking 2-3 minutes before understanding how gravity direction changes. This led to implementing an interactive tutorial mode that slows rotation initially and uses on-screen arrows showing current gravity direction. Also discovered that spectators found it difficult to follow gameplay from ground level‚Äîadded large score displays and player indicators that remain readable from any viewing angle. Most valuable feedback: players wanted competitive modes with escalating difficulty. Implemented "rally mode" for Pong where rotation speed increases with each successful hit, and "survival mode" for Tetris where gravity strength increases over time. These modes dramatically increased engagement and replayability.
